# 修復資料庫欄位錯誤

## 錯誤訊息
```
Could not find the 'collaboration_process' column of 'collaborations' in the schema cache
```

## 問題原因
資料庫的 `collaborations` 表格缺少以下欄位：
- `contract_status` - 合約狀態
- `collaboration_process` - 合作流程資訊

## 修復步驟

### 1. 開啟 Supabase SQL Editor
1. 前往 https://supabase.com/dashboard
2. 選擇您的專案
3. 點選左側選單的 **SQL Editor**

### 2. 執行更新 SQL
將 `SUPABASE_UPDATE_COLLABORATIONS.sql` 檔案的內容貼到 SQL Editor 中執行：

```sql
-- 1. 新增 contract_status 欄位
ALTER TABLE collaborations
ADD COLUMN IF NOT EXISTS contract_status VARCHAR(20)
CHECK (contract_status IN ('none', 'draft', 'pending_signature', 'signed', 'expired'))
DEFAULT 'none';

-- 2. 新增 collaboration_process 欄位（使用 JSONB 儲存結構化資料）
ALTER TABLE collaborations
ADD COLUMN IF NOT EXISTS collaboration_process JSONB DEFAULT '{}'::jsonb;

-- 3. 為現有資料設定預設值
UPDATE collaborations
SET contract_status = 'none'
WHERE contract_status IS NULL;

UPDATE collaborations
SET collaboration_process = '{}'::jsonb
WHERE collaboration_process IS NULL;
```

### 3. 執行並確認
1. 點擊 **Run** 按鈕執行 SQL
2. 確認顯示 "Success" 訊息
3. 重新整理您的網站

## 完成後
資料庫更新完成後，合作管理功能將正常運作，包括：
- 合約狀態管理
- 合作流程資訊顯示（洽談情況、商品寄送、合作細節等）
